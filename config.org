* How this file works

This is a literate configuration: the guide to the system and its
implementation live in one document. =init.el= is only a bootstrap  
it calls =org-babel-load-file= on this file, which extracts every
=emacs-lisp= block (in order) into =config.el= and loads it.
Re-tangling happens only when =config.org= is newer than =config.el=,
so startup stays fast after the first run.

To change the configuration: edit the blocks here, restart Emacs (or
=M-x org-babel-load-file= on this file). Never edit =config.el=   it
is generated. In a team, this file lives in the shared repository: the
conventions that must be universal (keyword set,
archive location) are distributed as code, not as a wiki page.

One standing rule from the leader-key mechanism: never bind anything
under =C-c= in this file   always bind under =M-m= directly.

* The One Round System

** Switching org directories (M-m O)

Use =M-m O= to switch between org directories (e.g., =~/org/= for private,
=~/nextcloud/org/= for work). Previously used directories are remembered
and offered for completion. The current choice persists across restarts.

** Dependencies and Gantt charts (org-edna)

Tasks gate each other through two properties, and every captured task
already carries the ID this needs (=M-x org-id-copy= on a heading puts
its ID on the clipboard):

- =:BLOCKER: ids(<id>)=   this task cannot be set DONE while the
  referenced task is still open. For simple chains, relative forms
  save the ID juggling: =:BLOCKER: previous-sibling= makes tasks
  depend on the one above them in the file.
- =:TRIGGER: ids(<id>) todo!(NEXT)=   when /this/ task is closed, the
  referenced task is automatically set to NEXT, so the next step
  surfaces in the agenda by itself. =next-sibling= works here too.

A typical project chain:

#+begin_src org
,* Aufgaben
,** TODO Pr?fstand aufbauen
SCHEDULED: <2026-06-15 Mo> DEADLINE: <2026-06-19 Fr>
:PROPERTIES:
:ID:      aaa-1111- 
:TRIGGER: next-sibling todo!(NEXT)
:END:
,** TODO Messreihe fahren
:PROPERTIES:
:BLOCKER: ids("aaa-1111- ")
:EFFORT:  3d
:END:
#+end_src

The Gantt chart falls straight out of this: =M-m p g= in a project
file builds a Mermaid gantt from every task. =SCHEDULED= is the start,
=DEADLINE= the end (otherwise =:EFFORT:= like "3d" sets the duration,
default one day); =BLOCKER ids( )= entries become =after=-dependencies,
so a task without its own SCHEDULED starts when its blockers end.
DONE/CANCELLED render greyed out, NEXT renders as active, priority
=[#A]= as critical, and top-level headings become sections. The export
writes =<project>.mmd= next to the org file and, when mermaid-cli is
installed (=npm i -g @mermaid-js/mermaid-cli=), renders and opens
=<project>.svg=.

** Time tracking   where the clock lives

Clock on the entity, never in the daily. =M-m i= on a task clocks in  
the clock lines land in /that task's/ =:LOGBOOK:=, exactly where the
rest of its history lives   =M-m o= clocks out, =M-m j= jumps to the
running clock, and =C-u M-m i= picks from recently clocked tasks, so
switching work is two keys from anywhere. The running clock shows in
the modeline and survives restarts.

Clocking into dailies would be an anti-pattern: it attributes time to
the /date/ instead of the /work/, and "wieviel Zeit hat Projekt X
gefressen" becomes unanswerable. The daily perspective on time is a
*view*, not a storage location: =M-m a d= shows today as a timeline of
all clocked intervals with a clock report on top   an automatically
written diary of the day. For per-project totals, drop a clocktable
into the project file and update it with =M-m M-c=:

#+begin_src org
,#+BEGIN: clocktable :scope file :maxlevel 3 :block thisweek :link t
,#+END:
#+end_src

If anything, this /reduces/ what dailies are for: the timeline of the
day writes itself, so the daily note stays purely narrative   what you
observed and thought, not what you did when.

** Working in a team

The repository is the team: one git repo holds the whole tree,
everyone clones it, pulls before the daily review, pushes after. The
one-file-per-project rule is also the merge-conflict strategy   people
on different projects never touch the same file, and when two people
do, logbook collisions are append-append conflicts (keep both lines,
done).

To hand a task to a colleague, set its =OWNER= property by hand (tasks
live with their project, not with their person); the colleague finds
it in their =M-m a m= ("My work") view after the next pull. Each person
sets =my/org-user= to match their git identity.

Colleagues without Emacs are degraded gracefully, not excluded: the
tree is plain text, any editor opens it, any project-wide search
replaces ripgrep, and the two human rules   never edit moment-records,
commit with a reason when you change an entity   are followable
in Notepad. The tooling automates the conventions; it doesn't define
them.

Two conventions no tool can enforce, agree on them explicitly: reasons
are written *for the stranger* (the colleague three years from now who
reads the commit message   "Lieferantenproblem" helps nobody; "M?ller
GmbH hat die 12mm-NBR-Variante eingestellt, keine Zweitquelle unter
5k MOQ" is gold), and the daily inbox review actually happens   an
inbox that silts up becomes a second workspace, and the system dies
of it.

** When to do what

| Situation                                                  | Do this                                                                    |
|------------------------------------------------------------+----------------------------------------------------------------------------|
| What happened today, context, half-thoughts                | Inbox                                                                      |
| Experiment                                                 | =M-m c l=   typed record in =records/=, project linked by ID                   |
| Meeting                                                    | =M-m c m=                                                                    |
| New task                                                   | capture, then refile (=M-m M-w=   picks any node via roam) into the project  |
| Hand a task to a colleague                                 | set =OWNER= by hand; appears in their My-work view                           |
| Task no longer relevant                                    | =M-m x a= to archive, or cut subtree and put in commit message in a git repo |
| Decision overruled / document replaced / parameter changed | cut the old node into the commit (Magit convention); write the new one     |
| Spec/BOM/protocol revision                                 | edit in place, bump =:REV:=; =git log -p= has the diff                         |
| "Didn't we already try this once?"                         | =M-m f g= from =~/org=   archives included; read the logbook you find          |

** The loop

Daily, ten minutes: pull; empty the inbox (refile or archive);
optionally write the daily note; push. Weekly, twenty minutes: sweep
WAIT and DLGT (anything stale gets a nudge or gets archived); archive
everything terminal whose dependencies are gone; skim ACTIVE decisions
for any that reality has silently overruled   cut them into a commit
/now/, while you still remember why.

** Why this stays discoverable

The commit message you write when cutting a dead-end task carries the
problem's own vocabulary, so future searches for the problem find the
abandoned attempt. The archives are ordinary org files under the same
root, so ripgrep and the org-roam graph reach them without knowing
they are archives   and because every record links its project by ID,
a project's backlink buffer lists every protocol and meeting ever
taken for it, even after the project itself was archived. Git is the
history layer: full-text diffs of large documents, the what / when /
whom / why of every change, disaster recovery, proof.

** Leader keys   M-m   C-c, M-o   C-x
M-m is bound to the *same keymap object* that C-c points to
(`mode-specific-map') and M-o to `ctl-x-map'. That way C-c/C-x can be
repurposed as copy/cut further down without losing the prefix maps, and
everything any package ever binds under C-c <letter> automatically shows
up under M-m. Mode-local C-c/C-x prefixes are folded in per buffer (see
the CUA section). MUST run before any `:bind ("M-m ...")' below.

#+begin_src emacs-lisp
(define-key global-map (kbd "M-m") mode-specific-map)
(define-key global-map (kbd "M-o") ctl-x-map)

;; RULE: never bind mode-local keys under "C-c ..." anywhere in this config
;; after this point   always bind them under "M-m ..." directly. A late
;; (define-key some-mode-map (kbd "C-c x")  ) would recreate a local C-c
;; prefix and confuse the remapping in the CUA section.

;; The two mechanism helpers live up here ON PURPOSE: packages that load
;; during init (dirvish pulls in dired, which pulls in wdired) run
;; `with-eval-after-load' blocks that call them immediately.

(defun my/mirror-c-to-m (keymap)
  "Add M-<key> bindings for every C-<key> binding in KEYMAP."
  (map-keymap
   (lambda (key def)
     (when (and (integerp key) (>= key ?\C-a) (<= key ?\C-z)
                (not (memq key '(?\C-g ?\C-h ?\C-i ?\C-j ?\C-m))))
       (let ((m-key (+ (- key ?\C-a) ?\M-a)))
         (unless (lookup-key keymap (vector m-key))
           ;; Skip keys that can't be mirrored: M-<k> is stored as ESC <k>,
           ;; which clashes when the map binds ESC itself to a command
           ;; (wdired does, for example).
           (ignore-errors
             (define-key keymap (vector m-key) def))))))
   keymap))

(defun my/minor-cua-fix (map)
  "Move MAP's C-c prefix bindings under M-m and free C-c.
MAP is a minor-mode keymap. Its C-c submap is mirrored (C-<k> -> M-<k>)
and exposed at M-m; sequences not in the submap fall through to the
major-mode and global leader maps."
  (let ((cc (lookup-key map (kbd "C-c"))))
    (when (keymapp cc)
      ;; If the submap binds ESC directly to a command (wdired: C-c ESC =
      ;; abort), every M-<k> mirror would clash with it, because Emacs
      ;; stores M-<k> as ESC <k>. The binding always duplicates C-c C-k,
      ;; so drop it before mirroring:
      (let ((esc (lookup-key cc (kbd "ESC"))))
        (when (and esc (not (keymapp esc)))
          (define-key cc (kbd "ESC") nil)))
      (my/mirror-c-to-m cc)
      (define-key map (kbd "M-m") cc)
      (define-key map (kbd "C-c") nil))))
#+end_src

*** Themes: professional dark/light pair with a switcher
#+begin_src emacs-lisp
;; Dark:  doom-dark+        port of VS Code's Dark+ (somber, desaturated)
;; Light: doom-one-light    clean, Office-like
;; M-m T toggles between them; the choice persists across restarts.
;; (doom-gruvbox stays installed   add it to `my/themes' to cycle it too.
;;  Built-in `modus-operandi' is another very professional light option.)
(defvar my/themes '(doom-dark+ doom-one-light)
  "Themes `my/theme-toggle' cycles through. First entry is the default.")
(defvar my/theme-file (expand-file-name "current-theme" user-emacs-directory)
  "File remembering the last active theme between sessions.")

(defun my/load-theme (theme)
  "Load THEME exclusively and remember it for the next session."
  (mapc #'disable-theme custom-enabled-themes)
  (load-theme theme t)
  (with-temp-file my/theme-file (insert (symbol-name theme))))

(defun my/theme-toggle ()
  "Cycle through `my/themes' (dark <-> light)."
  (interactive)
  (let* ((cur (car custom-enabled-themes))
         (next (or (cadr (memq cur my/themes)) (car my/themes))))
    (my/load-theme next)
    (message "Theme: %s" next)))

(defun my/saved-theme ()
  "Return the saved theme symbol, or the default (car of `my/themes')."
  (or (and (file-exists-p my/theme-file)
           (intern (string-trim
                    (with-temp-buffer
                      (insert-file-contents my/theme-file)
                      (buffer-string)))))
      (car my/themes)))

(defun my/apply-saved-theme (&optional frame)
  "Apply the saved theme   on FRAME if given (for daemon client frames)."
  (condition-case nil
      (let ((theme (my/saved-theme)))
        (if frame
            (with-selected-frame frame
              (when (display-graphic-p)
                (my/load-theme theme)))
          (my/load-theme theme)))
    (error (my/load-theme (car my/themes)))))

(defun my/apply-theme-on-first-gui-frame (frame)
  "Daemon only: apply the saved theme on the first GUI frame, then quit."
  (when (and (daemonp) (frame-live-p frame) (display-graphic-p frame))
    (my/apply-saved-theme frame)
    (remove-hook 'after-make-frame-functions
                 #'my/apply-theme-on-first-gui-frame)))

*** Live table alignment while typing
Org realigns a table only on =TAB= / =C-c C-c=. This realigns the
current table after each edit so columns stay lined up live. Two
subtleties make it safe: it is debounced onto a short idle timer so it
never fights fast typing, and it restores the cursor by =(row, column,
offset-within-field)= rather than by buffer position — because
=org-table-align= deletes and reinserts the whole table, a saved marker
would collapse to the table's top (that was the "cursor jumps to a new
line" bug). The offset is measured from the field's content start, so
the right-padding that alignment adds never drags the cursor along.

#+begin_src emacs-lisp
(defvar-local my/org-table-align-timer nil)

(defun my/org-table--field-offset ()
  "Chars from the current field's content start to point (>= 0)."
  (save-excursion
    (let ((p (point)))
      (skip-chars-backward "^|")   ;; back to just after the field's |
      (skip-chars-forward " ")     ;; forward to first content char
      (max 0 (- p (point))))))

(defun my/org-realign-current-table ()
  "Realign the table at point, keeping the cursor in the same field/spot."
  (setq my/org-table-align-timer nil)
  (when (and (derived-mode-p 'org-mode) (org-at-table-p))
    (let ((line   (line-number-at-pos))         ;; align never adds/removes rows
          (col    (org-table-current-column))
          (offset (my/org-table--field-offset))
          (inhibit-message t))
      (ignore-errors
        (org-table-align)
        (goto-char (point-min))
        (forward-line (1- line))
        (org-table-goto-column col)
        (skip-chars-forward " ")                ;; to content start
        (let ((field-end (save-excursion (skip-chars-forward "^|") (point))))
          (goto-char (min field-end (+ (point) offset))))))))

(defun my/org-table-align-maybe (&rest _)
  "Queue a debounced realign when point is inside a table."
  (when (and (derived-mode-p 'org-mode) (org-at-table-p))
    (when (timerp my/org-table-align-timer)
      (cancel-timer my/org-table-align-timer))
    (setq my/org-table-align-timer
          (run-with-idle-timer 0.5 nil #'my/org-realign-current-table))))

(defun my/org-realign-tables-on-edit ()
  "Enable live, cursor-preserving table realignment in this buffer."
  (add-hook 'post-self-insert-hook #'my/org-table-align-maybe nil t))
#+end_src

*** New .org files get #+title, #+date and an org id automatically
Visiting a brand-new empty =.org= file under =org-directory= inserts a
title (from the file name: dashes/underscores → spaces, capitalised; a
roam-style =20260612…-= timestamp prefix is stripped) plus the creation
date and a file-level org id. It only touches genuinely empty files and skips capture/roam
buffers (which write their own headers), so existing notes are never
disturbed. Capture is detected via a flag bound around the capture
commands, which stays set through the whole setup.

#+begin_src emacs-lisp
(defvar my/in-org-capture nil
  "Non-nil while an org capture is in progress, so my/org-new-file-header
knows to skip capture-created files (they write their own headers).")
(defun my/org-capture-wrap (orig &rest args)
  "Set `my/in-org-capture' around a capture command.
Uses setq with save/restore rather than a `let' on the special var:
let-shadowing a defvar'd symbol triggers a spurious \"defining as
dynamic an already lexical var\" warning under native compilation.
OLD is an ordinary lexical local, so it never collides. Save/restore
(vs. set-to-nil) also keeps nested captures correct."
  (let ((old my/in-org-capture))
    (setq my/in-org-capture t)
    (unwind-protect (apply orig args)
      (setq my/in-org-capture old))))
(require 'org-capture nil t)
(dolist (fn '(org-capture org-roam-capture org-roam-dailies-capture-today))
  (when (fboundp fn)
    (advice-add fn :around #'my/org-capture-wrap)))

(defun my/org-new-file-header ()
  "Insert #+title, #+date and a file-level org id into a fresh empty .org file.
Skips files created during a capture (which write their own headers),
detected via the `my/in-org-capture' flag bound around capture commands."
  (when (and buffer-file-name
             (string-suffix-p ".org" buffer-file-name)
             (not my/in-org-capture)
             (file-in-directory-p buffer-file-name (expand-file-name my/org-dir))
             (zerop (buffer-size)))
    (let* ((base  (file-name-base buffer-file-name))
           (clean (replace-regexp-in-string "\\`[0-9]\\{14\\}-" "" base))
           (title (capitalize (replace-regexp-in-string "[-_]+" " " clean))))
      (insert (format "#+title: %s\n#+date: %s\n\n"
                      title (format-time-string "[%Y-%m-%d %a]")))
      (save-excursion
        (goto-char (point-min))
        (org-id-get-create))
      (goto-char (point-max)))))
(add-hook 'org-mode-hook #'my/org-new-file-header)

*** Desktop reminders for due tasks
A timer scans the agenda 60 s after startup and every 30 min after,
firing one desktop notification per open task whose =DEADLINE= is within
24 h or already overdue. A session hash-table means each task is
reminded only once. =M-m R= scans on demand.

#+begin_src emacs-lisp
(use-package alert
  :custom
  (alert-default-style (cond ((eq system-type 'windows-nt) 'toaster)
                             ((eq system-type 'darwin)      'osx-notifier)
                             (t                             'libnotify))))

(defvar my/org-notified-deadlines (make-hash-table :test 'equal)
  "Tasks already notified this session, so we remind once.")

(defun my/org-deadline-reminders ()
  "Notify about tasks due within 24h or overdue (once each per session)."
  (interactive)
  (require 'alert)
  (let ((now (current-time)))
    (dolist (file (org-agenda-files))
      (when (file-exists-p file)
        (with-current-buffer (find-file-noselect file)
          (org-map-entries
           (lambda ()
             (let ((dl    (org-entry-get nil "DEADLINE"))
                   (state (org-get-todo-state)))
               (when (and dl (member state '("TODO" "NEXT" "WAIT" "DLGT")))
                 (let* ((due  (org-time-string-to-time dl))
                        (secs (float-time (time-subtract due now)))
                        (key  (or (org-id-get)
                                  (concat file "::" (org-get-heading t t t t)))))
                   (when (and (< secs 86400)
                              (not (gethash key my/org-notified-deadlines)))
                     (puthash key t my/org-notified-deadlines)
                     (alert (format "%s\n%s"
                                    (org-get-heading t t t t)
                                    (if (< secs 0) "OVERDUE"
                                      (format "due in %d h" (ceiling (/ secs 3600.0)))))
                            :title "Org deadline"
                            :severity (if (< secs 0) 'high 'normal)))))))
           t 'file))))))

(run-with-timer 60 (* 30 60) #'my/org-deadline-reminders)
(global-set-key (kbd "M-m R") #'my/org-deadline-reminders)

;; Every capture becomes a node. EVERY captured file gets a file-level ID
;; (idempotent: returns the existing one if present)   the :PROPERTIES:
;; drawer at the top makes the file a linkable roam node and lets the
;; links it contains (e.g. the record's "Projekt ::" line) register as
;; backlinks on their targets. ENTRY captures (meeting)
;; additionally get ID + CREATED on the heading; PLAIN file captures
;; (Laborprotokoll) also ID every TODO heading that came in via
;; the template. org-roam's own captures are skipped: roam manages those
;; IDs itself, and dailies would drown in property drawers.
(defun my/org-capture-stamp ()
  "Make the just-captured item a node (One Round System)."
  (unless (bound-and-true-p org-roam-capture--node)
    ;; Every file gets a file-level ID (the buffer is widened before this
    ;; hook runs, so point-min is the real file start):
    (save-excursion
      (goto-char (point-min))
      (org-id-get-create))
    (pcase (org-capture-get :type)
      ('entry
       (save-excursion
         (org-back-to-heading t)
         (org-id-get-create)
         (org-set-property "CREATED"
                           (format-time-string "[%Y-%m-%d %a %H:%M]"))))
      ('plain
       ;; TODO headings embedded in the template (e.g. a project's Aufgaben)
       ;; get IDs too, so every todo is linkable from birth:
       (org-map-entries
        (lambda ()
          (when (and (org-get-todo-state) (not (org-id-get)))
            (org-id-get-create)))
        nil 'file)))))
(add-hook 'org-capture-before-finalize-hook #'my/org-capture-stamp)

;; Link to a heading in ANOTHER file that isn't a node yet, without
;; leaving the current document. Flow: pick a recent .org file -> pick
;; one of its headings (completion; the file is opened invisibly, never
;; shown) -> that heading gets an ID if it lacks one, the file is saved,
;; and an [[id:…][Heading]] link is inserted back at the original cursor
;; position. A file we had to open ourselves is killed again afterwards;
;; a file you were already editing is left open (and unsaved buffers are
;; never force-closed).
(defun my/org-link-to-foreign-heading ()
  "Insert an id link to a heading in a recentf-picked org file."
  (interactive)
  (let* ((org-files (seq-filter (lambda (f) (string-suffix-p ".org" f))
                                recentf-list))
         (_ (unless org-files (user-error "No recent .org files")))
         (file (expand-file-name
                (completing-read "Datei: " org-files nil t)))
         (insert-at (point-marker))
         (already (find-buffer-visiting file))
         (buf (or already (find-file-noselect file)))
         link)
    (with-current-buffer buf
      (let* ((cands (org-map-entries
                     (lambda ()
                       (cons (format "%s%s"
                                     (make-string (* 2 (1- (org-current-level))) ?\s)
                                     (org-get-heading t t t t))
                             (point-marker)))
                     nil 'file))
             (_ (unless cands (user-error "No headings in %s"
                                          (file-name-nondirectory file))))
             (pick (completing-read "Header: " cands nil t))
             (mk   (cdr (assoc pick cands))))
        (save-excursion
          (goto-char mk)
          (let ((id (org-id-get-create))
                (title (org-get-heading t t t t)))
            ;; register the location now so the link resolves immediately,
            ;; without waiting for roam autosync / a global id scan
            (org-id-add-location id (buffer-file-name))
            (setq link (format "[[id:%s][%s]]" id title))))
        (when (buffer-modified-p) (save-buffer))))
    ;; only close the file if WE opened it and it has no pending edits
    (when (and (not already) (buffer-live-p buf)
               (not (buffer-modified-p buf)))
      (kill-buffer buf))
    (save-excursion
      (goto-char insert-at)
      (insert link))
    (goto-char (+ insert-at (length link)))))

(defun my/org-roam-node-insert-quick ()
  "Insert a roam link; new nodes use one no-choice template and open
for editing. Existing nodes are linked without any extra prompt."
  (interactive)
  (let* ((created nil)
         (grab (lambda ()
                 (setq created (buffer-file-name
                                (or (buffer-base-buffer) (current-buffer)))))))
    ;; org-roam-capture-templates is a defcustom in org-roam, which is
    ;; loaded lazily (first C-k). At compile time it isn't yet known to
    ;; be special, so a plain `let' would bind it LEXICALLY — the
    ;; override would be ignored and org-roam's later defcustom would
    ;; warn/err ("defining as dynamic an already lexical var"). `dlet'
    ;; forces a dynamic binding regardless of load order.
    (dlet ((org-roam-capture-templates
            '(("d" "default" plain "%?"
               :if-new (file+head
                        "${my/org-roam-capture-dir}${my/org-roam-node-slug-de}.org"
                        "#+title: ${title}\n#+date: %<[%Y-%m-%d %a]>\n\n")
               :immediate-finish t
               :unnarrowed t))))
      (add-hook 'org-roam-capture-new-node-hook grab)
      (unwind-protect (org-roam-node-insert)
        (remove-hook 'org-roam-capture-new-node-hook grab)))
    ;; if a new node was created, drop into it to write
    (when (and created (file-exists-p created))
      (find-file created)
      (goto-char (point-max)))))

#+begin_src emacs-lisp
;; Marginalia (and org-timeblock) call `seconds-to-string' with 3 args,
;; but the built-in subr only accepts 1. Override early so it accepts extras.
;; Redefine only when the current definition accepts FEWER than the 3 args
;; marginalia passes (newer Emacs versions already accept them natively).
;; time-date must be loaded first   on an autoload stub, `func-arity' errors:
(require 'time-date)
(let ((max-arity (cdr (func-arity (symbol-function 'seconds-to-string)))))
  (when (and (numberp max-arity) (< max-arity 3))
    (defun seconds-to-string (delay &rest _ignored)
    "Convert the time DELAY, in seconds, to a string."
    (cond ((< delay 0) (format "-%s" (seconds-to-string (- delay))))
          ((>= delay (* 1 60 60 24 365.25))
           (format "%.1fy" (/ delay (* 1 60 60 24 365.25))))
          ((>= delay (* 1 60 60 24 30.4375))
           (format "%.1fmo" (/ delay (* 1 60 60 24 30.4375))))
          ((>= delay (* 1 60 60 24))
           (format "%.1fd" (/ delay (* 1 60 60 24))))
          ((>= delay (* 1 60 60))
           (format "%.1fh" (/ delay (* 1 60 60))))
          ((>= delay (* 1 60))
           (format "%.1fm" (/ delay (* 1 60))))
          (t (format "%.1fs" delay))))))

** Gantt export: org + org-edna -> Mermaid
M-m p g in any org buffer. Builds a Mermaid gantt chart from all TODO
headlines: SCHEDULED = start, DEADLINE = end (else :EFFORT:, default 1d),
org-edna `:BLOCKER: ids(...)' entries become mermaid `after' dependencies,
DONE/NEXT/[#A] map to done/active/crit styling. Top-level headings become
sections. Renders to SVG if mmdc (mermaid-cli) is installed, otherwise
writes the .mmd source (also usable in a babel block / Marginalia app).

#+begin_src emacs-lisp
(defun my/org-gantt--date (ts)
  "Extract YYYY-MM-DD from an org timestamp string TS, or nil."
  (when (and ts (string-match "[0-9]\\{4\\}-[0-9]\\{2\\}-[0-9]\\{2\\}" ts))
    (match-string 0 ts)))

(defun my/org-gantt--days (effort)
  "EFFORT string (\"3d\", \"2:00\") -> whole days, minimum 1."
  (or (when effort
        (ignore-errors
          (max 1 (round (/ (org-duration-to-minutes effort) 1440.0)))))
      1))

(defun my/org-gantt--blockers (blocker)
  "Extract the list of ids from an org-edna BLOCKER property string."
  (when (and blocker (string-match "ids(\\([^)]*\\))" blocker))
    (split-string (match-string 1 blocker) "[ \t,\"]+" t)))

(defun my/org-gantt--clean (name)
  "Strip characters mermaid chokes on in task NAMEs."
  (replace-regexp-in-string "[:,#]" "-" name))

(defun my/org-mermaid-gantt ()
  "Return Mermaid gantt source for all TODO headlines in the current buffer."
  (require 'org-duration)
  (let ((ids (make-hash-table :test #'equal)) ; org ID/CUSTOM_ID -> mermaid id
        (n 0) entries)
    ;; Pass 1: collect tasks, assign short mermaid ids.
    (org-map-entries
     (lambda ()
       ;; Tasks only: the artifact states (ACTIVE/ONHOLD/FINISHED/
       ;; SUPERSEDED/DROPPED   decisions, documents, processes) carry no
       ;; schedule semantics and stay out of the gantt.
       (when (member (org-get-todo-state)
                     '("TODO" "NEXT" "WAIT" "DLGT" "DONE" "CANCELLED"))
         (let* ((oid (or (org-entry-get nil "ID")
                         (org-entry-get nil "CUSTOM_ID")))
                (mid (format "t%d" (setq n (1+ n)))))
           (when oid (puthash oid mid ids))
           (push (list :mid mid
                       :name (my/org-gantt--clean (org-get-heading t t t t))
                       :todo (org-get-todo-state)
                       :prio (org-entry-get nil "PRIORITY")
                       :sched (my/org-gantt--date (org-entry-get nil "SCHEDULED"))
                       :dead  (my/org-gantt--date (org-entry-get nil "DEADLINE"))
                       :days  (my/org-gantt--days (org-entry-get nil "EFFORT"))
                       :deps  (my/org-gantt--blockers (org-entry-get nil "BLOCKER"))
                       :section (or (car (org-get-outline-path)) "Tasks"))
                 entries))))
     t nil)
    (setq entries (nreverse entries))
    ;; Pass 2: emit.
    (let ((title (my/org-gantt--clean
                  (or (cadr (assoc "TITLE" (org-collect-keywords '("TITLE"))))
                      (and buffer-file-name (file-name-base buffer-file-name))
                      "Project"))))
      (with-temp-buffer
        (insert "gantt\n"
                "  dateFormat YYYY-MM-DD\n"
                (format "  title %s\n" title))
        (let ((cur-section nil))
          (dolist (e entries)
            (unless (equal cur-section (plist-get e :section))
              (setq cur-section (plist-get e :section))
              (insert (format "  section %s\n" (my/org-gantt--clean cur-section))))
            (let* ((todo (plist-get e :todo))
                   (flags (concat
                           (cond ((member todo '("DONE" "CANCELLED")) "done, ")
                                 ((equal todo "NEXT") "active, ")
                                 (t ""))
                           (when (equal (plist-get e :prio) "A") "crit, ")))
                   (deps (delq nil (mapcar (lambda (d) (gethash d ids))
                                           (plist-get e :deps))))
                   (start (cond ((plist-get e :sched) (plist-get e :sched))
                                (deps (concat "after " (mapconcat #'identity deps " ")))
                                (t (format-time-string "%Y-%m-%d"))))
                   (end (or (plist-get e :dead)
                            (format "%dd" (plist-get e :days)))))
              (insert (format "  %s :%s%s, %s, %s\n"
                              (plist-get e :name) (or flags "")
                              (plist-get e :mid) start end)))))
        (buffer-string)))))

** Per-file commit + history mirror
A file-scoped commit flow alongside Magit. =C-S-s= saves the current
buffer, stages its file, and opens a commit-message buffer (C-s finalize,
C-w abort); on finalize it commits and writes a read-only org record

message and the diff, named =<file>--<timestamp>--<hash>.org=. =M-m f h=

(defun my/save-stage-commit ()
  "Save the current buffer, stage the file, and open a commit-message buffer."
  (interactive)
  (unless buffer-file-name (user-error "Buffer has no file"))
  (save-buffer)
  (let* ((file buffer-file-name)
         (root (my/current-repo-root))

  :lighter " Commit"
  :keymap (let ((map (make-sparse-keymap)))
            (define-key map (kbd "C-s") #'my/finalize-commit)
            (define-key map (kbd "C-w") #'my/abort-commit)
            map))

(defun my/abort-commit ()
  "Abort the pending commit. File remains staged."
  (interactive)
  (quit-window t)
  (message "Commit aborted (file stays staged)"))

(defun my/finalize-commit ()
  "Commit the staged file using the current buffer as the message,
then write a read-only history record with the diff."
  (interactive)
  (when (string-blank-p (buffer-string)) (user-error "Empty commit message"))
  (let* ((file my/pending-commit-file)
         (root my/pending-commit-root)
         (msg (buffer-substring-no-properties (point-min) (point-max)))
         (default-directory root))
    (call-process-region (point-min) (point-max) "git" nil nil nil "commit" "-F" "-")
    (quit-window t)
    (my/write-history-record file root msg)
    (message "Committed and archived.")))

(defun my/write-history-record (file root msg)
  "Write a read-only org file capturing the commit MSG and diff for FILE."
  (let* ((default-directory root)
         (history-dir (my/history-dir root))
         (hash (string-trim (shell-command-to-string "git rev-parse HEAD")))
         (rel (file-relative-name file root))
         (diff (shell-command-to-string
                (format "git show --format= %s -- %s"
                        (shell-quote-argument hash)
                        (shell-quote-argument rel))))
         (uuid (org-id-new))
         (target (expand-file-name
                  (format "%s--%s--%s.org"
                          (my/slugify rel)
                          (format-time-string "%Y%m%dT%H%M%S")
                          (substring hash 0 8))
                  history-dir)))
    (unless (file-directory-p history-dir)
      (make-directory history-dir t))
    (with-temp-file target
      (insert (format ":PROPERTIES:\n:ID: %s\n:GIT_HASH: %s\n:GIT_FILE: %s\n:GIT_DATE: %s\n:END:\n\n"
                      uuid hash rel (format-time-string "%FT%T%z")))
      (insert msg "\n\n")
      (insert "#+begin_src diff\n" diff "#+end_src\n"))
    (set-file-modes target #o444)))

(defun my/browse-file-history ()
  "Browse history records for the current buffer's file, newest first."
  (interactive)
  (let* ((root (my/current-repo-root))
         (rel (file-relative-name buffer-file-name root))
         (slug (my/slugify rel))
         (candidates (sort (directory-files (my/history-dir root) t (concat "^" (regexp-quote slug) "--"))
                           #'string>)))
    (unless candidates (user-error "No history for this file"))
    (find-file (completing-read "History: " candidates))
    (view-mode 1)))

(global-set-key (kbd "C-S-s") #'my/save-stage-commit)
(global-set-key (kbd "M-m f h") #'my/browse-file-history)
#+end_src
